Did I understand correctly that you ended up using the Pan_GI_L2 clustering in the end for DE analyses? The report and the Seurat file also contain ‘cluster’ clustering where there are, for example, two CD4 and two CD8 populations. I would just want to hear your thoughts on why one is better than the other.
 

Yes, that’s correct. For the differential expression analyses, we used the Pan-GI_L2 annotation, rather than the Monaco-based annotation.

 

The Pan-GI atlas was built from hundreds of normal and diseased gastrointestinal samples, capturing tissue-resident immune states in the context of gut inflammation and disease. This work is described in a recent large-scale study published in Nature (https://www.nature.com/articles/s41586-024-07571-1), which integrated over 1.6 million cells across more than 25 studies spanning multiple regions of the gastrointestinal tract. Because this reference is derived directly from gut tissue—including disease contexts similar to our samples, it provides a high-quality and biologically appropriate framework for defining cell identities in duodenal biopsies.

 

The Monaco reference is designed to resolve immune cell states in greater detail and is typically very useful in blood/PBMC-focused datasets. However, because this dataset consists of duodenal biopsies, we chose to prioritize the Pan-GI atlas, which is tissue-specific and better reflects gut-resident immune and epithelial cell states.

 

When we examined both annotations in more detail, the cell-type proportions inferred using the Pan-GI annotation were biologically consistent with expectations, whereas the proportions obtained using the Monaco annotation were less interpretable. This likely reflects the multiple mismatches between a circulating immune reference and a tissue-resident dataset.

 

Could you share with me the marker genes for (maybe both) Pan_GI_L2 and ‘cluster’ clustering? If I understood correctly, they will use them to design the Xenium analysis and probably know which one of these is better for this particular purpose.

Major changes from the draft we discussed back in January 2024 draw from experience we have gained in other projects since that time, in addition to consideration of how best use the additional material from these trial samples compared to the pilots run throughout 2023.  These changes are:

A major concern is cell yield from these samples, as only 1 of the 3 pilots yielded sufficient cells for scRNA-seq.  Consequently, we would like to use all of the 3 vials available for each of these trial samples, but also keen to avoid using all of this irreplaceable primary sample at one time.  We propose to use 1 of the 3 vials available for each of the 8 clinical trial samples, and repeat this on 3 subsequent days.  This will make the benchwork easier on each individual day, and mitigate the non-zero risk of technical failure during sort or capture on any given day.  After cell capture libraries can be combined for sequencing and analysis.
On each day we will have 2 separate pools, where the 4 trial subjects can be combined with differentiation by genotype, but to isolate the 2 timepoints for each subject those will be handled in parallel in 2 pools of 4 samples.  To avoid these pools being a direct confounder of our primary comparison of pre vs. post intervention, we propose to scramble the timepoints across pools.  Ie. pool 1 is S1 T1, S2 T1, S3 T2, S4 T2
To measure and potentially adjust for differences between the 2 pools on any given day we propose to include an identical spike in both pools from a duodenal biopsy of a 5th donor.  This will require a smaller amount of sample from a 5th donor on each of the 3 experimental days, for which genome sequencing is available that we confirm to support demultiplexing from the 4 trial subjects before going ahead.

Hi Lucas,

 

Thanks for sending these comments on the pilot work in duodenal samples. 

 

We seem to be drawing different conclusions about how the approach demonstrated in pilot 3 might be applied to clinical trial samples?  For the 3 pilots to date initial single cell preparations after digestion have had live cell counts of: 6x10e4, 2x10e5, 2x10e6.  Only the yield from the last of those is compatible with generating scRNA-seq if using the workflow demonstrated for pilot 3 and analysing samples separately.  Our suggestion is that the workflow used in pilot 3 could generate data for duodenal samples from most subjects if modified to include pooling of subjects. 

 

This is because in pilot 3 we obtained 2x10e6 cells at the initial stage of single cell preparation after digest, and used all of this to flow sort enough cells for a post sort AOPI count that established scRNA-seq would be possible.  That post sort AOPI count was 2x10e5 cells at 80% viability, of which only 17k cells were loaded onto the chromium as there was only 1 sample in pilot 3.  In our proposal for the trial samples (attached) we suggest to again sort from 2x10e6 cells after initial digest, but now to generate this by pooling 400k from each of the 5 patients.  This would enable us to use the same workflow where a post sort AOPI count is possible prior to chromium loading, but here 85k cells would be loaded on the chromium based on 17k x 5 samples.  Pooling subjects and demultiplexing the single cell data based on SNPs is routine, we typically do this to enable AOPI counts just prior to chromium loading which are aggregates of cells pooled from multiple donors, and also because processing the samples together minimizes batch effects.  This plan could be duplicated to run the pre and post treatment samples in parallel, each comprising 5 subjects and sorting from 2x10e6 pooled cells. 

 

Even if using this pooling strategy 4x10e5 cells/sample would be required at the live cell count after initial digestion, and this was obtained for only 1 out of the 3 pilot samples to date.  However the counts at this stage for those 3 pilots (2x10e5, 6x10e4, 2x10e6) are not so far off that it should be possible to obtain data for most.  Our thought is to run an initial experiment with only 1 of the 3 biopsy samples from each, preserving the remainder for subsequent expts if required. 

 

There are certainly other possible approaches but please let us know your thoughts given this rationale?
